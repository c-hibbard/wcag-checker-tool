WCAG 2.2 Accessibility Checker

A tiny, local-only checker for missing alt, inputs without labels, and low contrast. Ships as a Chrome/Edge extension—no servers, no accounts.

Open on GitHub Install (Load Unpacked) See features Privacy

Tip: If you haven’t used unpacked extensions before, the steps below take ~1 minute.

Install the extension (unpacked)

  1. Download or clone the repo.
  2. Open chrome://extensions (or edge://extensions).
  3. Toggle Developer mode.
  4. Click Load unpacked → select the extension/ folder (it contains manifest.json).
  5. Pin the toolbar icon. Optional: set a shortcut at chrome://extensions/shortcuts.

Extensions can’t run on chrome://* pages or the Chrome Web Store. Test on any normal website.

What it does

Checks

  • Alt text on images (WCAG 1.1.1)
  • Labels for form inputs (WCAG 3.3.2, 1.3.1)
  • Contrast for visible text (WCAG 1.4.3), with 3.0:1 for large/bold and 4.5:1 for normal text

Overlay tools

  • Filters: Alt / Labels / Contrast
  • Only interactive text (links, buttons, inputs)
  • Per-item Hide/Unhide & Show muted
  • Copy selector & WCAG references per item
  • Export JSON report for tickets

Ignore in your HTML

Add data-wcag-ignore to suppress checks in a subtree:

<div data-wcag-ignore="all">...</div>
<div data-wcag-ignore="contrast,labels">...</div>

Notes & limitations

  • Does not scan chrome://* pages or the Web Store (browser restriction).
  • Contrast on image/gradient backgrounds is skipped to reduce false positives.
  • The shown selector is a concise “find-it-fast” path, not a guaranteed unique selector.
